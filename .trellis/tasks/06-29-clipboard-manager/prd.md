# PRD: Rust Tauri Windows 剪切板管理工具

## Goal

创建一个基于 Rust Tauri 的 Windows 剪切板管理工具，支持剪切板历史记录、搜索功能，以及通过将 SQLite 数据库文件放置在同步盘（如 OneDrive）实现多设备非实时同步。

## Background

用户需要一个高效的剪切板管理工具来：
- 记录和检索剪切板历史
- 在多台 Windows 设备间同步剪切板内容
- 使用 UUID v7 作为记录 ID 以保证时序性和唯一性
- 确保工具运行效率，不影响系统性能

## User Requests (As-Is)

"从零开始 创建一个rust tarui 的windows剪切板管理工具 包含搜索 多设备非实时同步(直接将sqlite文件放到类似onedrive的同步盘) id用uuidv7 需要考虑效率"

## Confirmed Facts

- **技术栈**: Rust + Tauri（跨平台桌面应用框架）
- **目标平台**: Windows
- **核心功能**: 剪切板监控、历史记录、搜索
- **同步方案**: 将 SQLite 数据库文件放置在 OneDrive 等同步盘，实现多设备非实时同步
- **ID 策略**: 使用 UUID v7（基于时间戳，保证时序性和唯一性）
- **效率要求**: 需要考虑效率，不能影响系统性能

## Requirements

### 功能需求

1. **剪切板监控**
   - 实时监控系统剪切板变化
   - 自动记录剪切板内容（文本、图片等）
   - 支持多种数据格式

2. **历史记录管理**
   - 存储剪切板历史记录
   - 支持按时间排序
   - 支持删除单条或批量删除
   - 支持收藏/置顶功能
   - 默认保留 10000 条记录，可配置为不限制

3. **搜索功能**
   - 全文搜索剪切板内容
   - 支持按时间范围筛选
   - 支持按内容类型筛选

4. **多设备同步**
   - SQLite 数据库文件可放置在 OneDrive 同步盘
   - 多设备间自动同步（依赖同步盘的同步机制）
   - 处理同步冲突（UUID v7 基于时间戳，可按时间排序）

5. **数据格式支持**
   - 纯文本
   - 富文本（RTF）
   - 图片（PNG、JPEG 等）- 仅保存文件路径，不存 BLOB
   - 文件路径

### 非功能需求

1. **性能要求**
   - 剪切板监控延迟 < 100ms
   - 搜索响应时间 < 500ms
   - 内存占用 < 100MB
   - CPU 占用 < 5%（空闲时）

2. **数据安全**
   - 不加密（保持简单和高性能）

3. **用户体验**
   - 系统托盘常驻
   - 全局快捷键呼出主界面（默认 `Ctrl+W`，支持自定义）
   - 简洁直观的界面
   - 托盘右键菜单：显示主窗口、最近 5 条记录（快速粘贴）、设置、退出

## Technical Notes

### UUID v7 优势
- 基于时间戳生成，天然支持时序排序
- 在分布式/多设备环境下保证唯一性
- 适合作为主键，提高 SQLite 查询效率

### SQLite 同步方案
- 将 SQLite 文件放在 OneDrive 同步目录
- 依赖 OneDrive 的文件同步机制
- 需要考虑：
  - 写入冲突处理（UUID v7 减少冲突概率）
  - 数据库锁定问题（Tauri 应用需正确处理 SQLite 锁）
  - 同步延迟（非实时，用户需理解）

### 技术栈
- **后端**: Rust + Tauri
- **前端**: React + TypeScript + Tailwind CSS + Vite
- **数据库**: SQLite（通过 rusqlite 或 sqlx）
- **UUID**: uuid crate（v7 版本）
